# タスク管理アプリ 要件定義書

## 1. 概要・目的

Trello風のカンバン方式で個人のタスクを管理するWebアプリケーションを開発する。  
自分自身のタスク管理ツールとして日常的に利用することを目的とする。

## 2. 利用者・利用環境

- **利用者**: 自分ひとり（単一ユーザー想定、認証機能なし）
- **利用環境**: PCブラウザ（同一ブラウザでの継続利用を想定）

## 3. 機能要件

### 3.1 画面構成
- 画面は1画面のみ（ボード切り替え機能なし）
- 画面上に「未着手 / 作業中 / 完了」の3カラムを横並びで常時表示する
- カラムは固定（追加・削除・名前変更は行わない）

### 3.2 カード（タスク）機能
カードは以下の属性を持つ：

| 属性 | 内容 |
|---|---|
| タイトル | 必須。タスク名 |
| 説明 | 任意。詳細情報 |
| 優先度 | 高 / 中 / 低 から選択 |
| 期限 | 任意。日付 |

操作：
- カードの新規作成（任意のカラムに追加）
- カードの編集（全属性を変更可能）
- カードの削除

### 3.3 ドラッグ＆ドロップ機能
- カードを **別カラム** へドラッグ＆ドロップで移動できる
- カードを **同一カラム内** でドラッグ＆ドロップで並び替えできる

### 3.4 データ永続化
- すべてのデータはブラウザの localStorage に保存する
- ブラウザを閉じて再度開いてもデータが保持される

## 4. 非機能要件

- **パフォーマンス**: カード数が数百件程度でも快適に動作すること
- **対応ブラウザ**: 最新版の Chrome / Edge / Safari
- **レスポンシブ対応**: 今回は対象外（PC画面のみ想定）
- **セキュリティ**: ローカル保存のみのため特別な要件なし

## 5. データ構造（案）

```ts
type Priority = "high" | "medium" | "low";
type ColumnId = "todo" | "doing" | "done";

type Card = {
  id: string;
  title: string;
  description: string;
  priority: Priority;
  dueDate: string | null; // ISO形式
  columnId: ColumnId;
  order: number; // 同一カラム内の並び順
};

// localStorage に Card[] を JSON で保存
```

## 7. 対象外機能（スコープ外）

今回は以下の機能は **作らない**：

- ユーザー認証・ログイン
- 複数ボード対応
- カラムのカスタマイズ
- 複数人での共有・リアルタイム同期
- ラベル・タグ・添付ファイル
- コメント機能
- 検索・フィルタ
- 通知・リマインダー
- モバイル対応

## 8. 技術スタック（予定）

| スタイリング | Tailwind CSS（予定） |
| データ保存 | ブラウザの localStorage |
